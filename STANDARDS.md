# Standards

Non-negotiable standards for `jbcom/pkgs`. Agents and humans both follow
these; CI enforces as much as possible.

## Code

- **Node 24 LTS**. No other Node versions are supported locally or in CI.
- **pnpm 10** with `pnpm-lock.yaml` committed. `--frozen-lockfile` in CI.
- **Astro 5.x**, Vue 3, Tailwind 4.x. Do not introduce React.
- **TypeScript** for any new `.ts`/`.astro` logic. No `any`.
- **Prettier** default config; run `pnpm format` before committing.
- **No `console.log` shipped to production.** Keep `console.error` for
  genuine error-path reporting.
- **Max 300 LOC per source file.** Split larger files by responsibility.

## Package manifests

- Never hand-edit `Formula/*.rb`, `bucket/*.json`, or
  `choco/**/*.nuspec`. They are generated by upstream release pipelines.
- If you must seed one (e.g., for initial setup), mark it clearly with
  a comment: `# This file was generated by GoReleaser. DO NOT EDIT.`
- Validate locally before committing: `ruby -c Formula/*.rb` +
  the Python snippets in `README.md` for Scoop/Chocolatey.

## Actions & CI

- **Every action pinned to full commit SHA** with a tagged comment:

  ```yaml
  uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2
  ```

- **No `@main`** refs for actions.
- **No `workflow_call` jobs** without explicit `permissions:` declarations.
- **Dependabot** maintains SHAs weekly (Mondays 05:00 America/Chicago).
  Automerge workflow merges non-major bumps unattended.

## Commits

- **Conventional Commits**: `feat:`, `fix:`, `chore:`, `docs:`,
  `refactor:`, `perf:`, `test:`, `ci:`, `build:`, `revert:`.
- **Squash-merge PRs**. Comprehensive branch histories are fine — the
  squash title ends up in the changelog via release-please.
- **`git add -A` must be safe** — if not, fix `.gitignore` first.

## Documentation

Every markdown file in the repo root and under `docs/` has YAML
frontmatter with at least these fields:

```yaml
---
title: <human-readable title>
updated: <YYYY-MM-DD>
status: current    # current | draft | stale | archived
domain: <one-of>   # technical | product | quality | ops | creative | context
---
```

Pages without frontmatter are considered stale and get updated on
next pass.

## Brand

- Site accent: `#7b3f99` (brand purple — favicon + logo)
- Primary font: Fontsource Gabarito (variable); fallback system-ui
- No emojis in code. Emojis OK in user-facing copy only where they
  meaningfully distinguish content (e.g., packager tags 🍺 🪣 🍫).

## Accessibility

- Every non-decorative `<img>` has descriptive `alt` text
- Interactive elements reachable via keyboard
- Dark mode + light mode both tested (system + manual toggle)
- Lighthouse accessibility score ≥ 95 on the landing page
